JSX

JSX是什么

JSX并不是字符串或者HTML，它是JavaScript的一种符号扩展。另外，JSX也是一种表达式，其执行结果是React Component，其本质是JavaScript object。
JSX经过babel编译时通过执行React.createElement()生成React Component对象，其过程大致如下：
正式由于所有JSX在babel编译时必须调用React.createElement()，因此React库必须存在在你的作用域中。

因此，我们可以将JSX作为参数传递给函数，也可将其作为函数返回结果，并可将之赋值给变量。

由于JSX比HTML更接近JavaScript，因此JSX中标签的属性名是驼峰式的命名，例如：className表示的是html中标签属性名称。

预防注入式攻击

React DOM 在渲染之前默认会过滤所有传入的值。它可以确保你的应用不会被注入攻击。所有的内容在渲染之前都被转换成了字符串。这样可以有效地防止
<a href="http://www.imooc.com/article/13553" onclick="console.log(' a is clicked'); return false;">XSS(跨站脚本) 攻击。</a>

无论何种XXS攻击都需要通过服务器将攻击代码传递给用户，而React DOM在页面渲染之前将所有标签嵌入内容都转换成字符串了
意识是说，React DOM会将所有传入值(JSX中传入的变量)，进行编码转义后再输出。例如用户评论区中的内容以变量content表示，React DOM会把content中的所有可疑符号如<转换成HTML实体 &It后再进行渲染

当组件的props值为字符串时，字符串会先进行HTML转移。
