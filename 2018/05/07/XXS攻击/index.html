XSS介绍

XSS是什么？

Cross-site-scripting 的缩写，又称跨站脚本攻击，是客户端脚本注入的一种，主要指攻击者在页面植入恶意代码，当用户浏览到写入JavaScript之时，被嵌入Web网页里面的JavaScript恶意代码就会自动被执行，从而达到恶意攻击用户的目的，例如盗取各类用户帐号、网站挂马、盗窃企业重要信息等。

XSS是一种注入式攻击，简单来说用户A在页面输入恶意内容并提交，这些恶意代码运行在B用户打开的网页上了。

XSS例子说明

一个评论系统页面，用户A在评论区提价内容：%3Cscript%3Econsole.log(document.cookie)%3C/script%3E，则这些内容提交到服务器，如果服务器并不对A提交的内容做任何处理便返回给浏览器，则浏览器渲染的HTML内容可能如下：



    
      %3Chtml%3E
        …
        

          // 恶意代码
        

        …
      %3C/html%3E

这样，用户B访问该页面时，其cookie信息的控制权就完全交到恶意代码的手中

如何预防XSS

从XSS成因方面分析解决方案



  • 后端模板问题

    如果后端模板没有对HTML插入内容做任何处理，便容易造成XSS攻击漏洞，例如：




            评论：<?php echo $content; ?>


    这里，若对用户输入内容$content不做任何处理直接输出便形成漏洞^..^


    解决方法&nbsp只需要后台输出的时候，将可疑的符号 < 符号变成 < （HTML实体）就行。




  • 前端代码问题

    前端写页面时直接将内容拼接处来：




            $p.html(content)
            $p = $(‘
    ‘+ content +’
    ‘)


    这里，内容content不做任何处理直接输出渲染，game over^..^


    解决方法&nbsp不要自己拼 HTML，尽量使用 text 方法。如果一定要使用 HTML，就把可疑符号变成 HTML 实体。
